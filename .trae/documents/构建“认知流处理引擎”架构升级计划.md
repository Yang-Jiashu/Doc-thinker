# 架构升级：构建“认知流处理引擎” (Cognitive Stream Engine)

## 目标
将系统从被动的 RAG 工具升级为主动的“第二大脑”，实现对任意输入流的**实时理解、关联与推理**，通过 Test Time Compute（推理时计算）来解放人类脑力。

## 实施路线图

### Phase 1: 全形态信息流接入与认知管道 (The Cognitive Pipeline)
**核心任务**：建立能够处理任意输入的通用接口，并引入“写入时思考”机制。
1.  **通用接入层 (Universal Ingestion)**
    *   重构 API，不再局限于文件。新增 `IngestRequest` 模型，支持 Text, URL, Blob 等多种输入。
    *   实现 `StreamProcessor`，作为所有信息的统一入口。
2.  **认知处理器 (Cognitive Processor)**
    *   **理解 (Understand)**：对输入流进行语义分析，生成摘要和标签。
    *   **关联 (Associate)**：在写入前，**主动检索**现有图谱（Global Graph），寻找潜在连接（如：“这个 'Transformer' 和我上次读的 'Attention Is All You Need' 是同一个概念”）。
    *   **双写机制升级**：将经过“认知处理”后的结构化知识（不仅仅是原文）同时写入 Session 和 Global Graph。

### Phase 2: 复杂推理与任务拆解 (Reasoning & Decomposition)
**核心任务**：利用 `auto_thinking` 模块实现对复杂信息的自动拆解。
1.  **任务拆解器 (Task Decomposer)**
    *   将 `QuestionDecomposer` 泛化为 `InformationDecomposer`。
    *   遇到复杂输入时，自动拆解为多个子知识点。
2.  **Test Time Compute 循环**
    *   实现“思考循环”：对于模糊或复杂的输入，系统自主生成“追问”（Self-Query），自我完善上下文后再存储。

### Phase 3: 多模态超图的可视化与交互
**核心任务**：让“大脑”的运作透明化。
1.  **实时神经元可视化**：前端展示新知识如何连接到旧知识网络。
2.  **知识编辑**：允许用户手动修正系统的错误关联。
