# NeuroAgent - ToC 个人知识助手产品架构

## 产品定位

**NeuroAgent** 是一个面向个人用户的智能知识助手，核心能力是通过理解用户的文档、笔记和对话，自动构建个人知识图谱，并提供智能联想和检索。

**核心差异**：不是文档管理工具，而是**会联想的第二大脑**。

---

## 核心创新

### 1. 文档即记忆输入 (Document as Memory Input)

在 ToC 场景中，用户上传的文档是知识输入的主要方式：

```
用户行为          系统处理
─────────────────────────────────────────
上传论文        →  提取层级结构
记录笔记        →  识别核心概念
保存文章        →  建立概念关联
日常对话        →  形成情境记忆

所有输入 → Episode → 层级化 KG → 自动联想网络
```

**关键洞察**：用户不擅长手动整理知识标签，系统需要自动理解文档结构。

### 2. 层级化知识图谱 (Hierarchical KG)

**三层架构**：

```
Level 3 (高阶抽象)          Level 2 (中阶概念)          Level 1 (低阶具体)
┌──────────────┐           ┌──────────────┐           ┌──────────────┐
│   Domain     │           │   Concept    │           │  Episode     │
│   (领域)      │◄─────────│   (概念)      │◄─────────│  (实例)       │
│              │  contains │              │ instance_of│              │
│  人工智能     │           │  深度学习     │           │  某篇论文     │
│  医学        │           │  CNN        │           │  一次对话     │
│  法律        │           │  Transformer│           │  一个笔记     │
└──────────────┘           └──────────────┘           └──────────────┘
         │                          │                        │
         └──────────────────────────┴────────────────────────┘
                              ▲
                              │
                    自动联想 (双向流动)
```

**双向联想**：
- **向上抽象**：从具体文档归纳所属领域
  - "这篇论文 → 关于深度学习 → 属于人工智能"
- **向下具体化**：从概念找到相关实例
  - "人工智能 → 包含深度学习 → 相关论文有..."

### 3. 自动联想生成新边/节点

**不是静态图谱，而是动态生长的记忆网络**：

```
新文档 "Transformer 论文" 加入
         │
         ▼
┌────────────────────────────────────────┐
│  1. 提取概念节点: [Transformer, Attention] │
│  2. 识别所属领域: "人工智能"               │
│  3. 发现已有概念: "深度学习"               │
│  4. 自动建立关联:                          │
│     doc_transformer ──[同主题]──▶ doc_dl   │
│     Transformer ──[属于]──▶ 人工智能       │
│  5. 跨文档联想:                            │
│     "这与你的 CNN 笔记相关"                │
└────────────────────────────────────────┘
```

**联想触发时机**：
1. **On-Insert**：新记忆写入时立即联想
2. **Background**：后台巩固时发现潜在关联
3. **Query-Time**：查询时扩散激活相关记忆
4. **Spontaneous**：被动监听关键词，主动浮现记忆

## 产品功能场景

### 场景 1: 智能学习助手

```
用户上传 10 篇 AI 论文
         │
         ▼
系统自动:
  - 识别领域: "人工智能"
  - 提取概念: ["CNN", "RNN", "Transformer", "GAN"]
  - 建立层级:
      人工智能
      ├── 计算机视觉
      │   └── CNN (3篇论文)
      └── 自然语言处理
          └── Transformer (2篇论文)
  - 发现关联:
      "这 3 篇 CNN 论文都提到了 ResNet"
  - 学习建议:
      "建议先阅读 CNN 基础，再看 Transformer"
```

### 场景 2: 会议/笔记智能关联

```
周一会议记录: "讨论 Q3 营销方案"
周三笔记: "竞品分析报告"
周五文档: "预算规划表"
         │
         ▼
系统自动发现:
  - 都包含关键词 "Q3"、"预算"
  - 建议建立 "Q3 规划" 主题
  - 下次提到 "Q3" 时自动关联所有相关笔记
```

### 场景 3: 灵感联想

```
用户正在写文案，卡住了
         │
         ▼
系统检测到用户输入 "品牌定位"
         │
         ▼
自动联想:
  - 上周保存的文章 "Apple 品牌策略"
  - 之前的笔记 "目标用户分析"
  - 相关概念 "差异化竞争"
         │
         ▼
主动提示: "你之前收藏的 Apple 案例分析可能有用"
```

### 场景 4: 知识盲区提醒

```
用户经常查看 "深度学习" 相关内容
但从未涉及 "强化学习"
         │
         ▼
系统建议:
  "你关注深度学习很久了，
   要不要了解一下强化学习？
   这是同一领域的互补技术。"
```

## 技术架构

```
用户交互层 (UI/App)
├── 文档上传
├── 对话界面
├── 知识图谱可视化
└── 记忆检索界面

感知层 (Perception)
├── DocumentPerceiver (文档感知)
│   ├── PDF/Word/Markdown 解析
│   ├── 层级结构提取
│   └── 概念自动识别
├── ChatPerceiver (对话感知)
└── APIPerceiver (外部数据)

记忆核心层 (Neuro Core)
├── HierarchicalKG (层级化 KG)
│   ├── 三层节点管理
│   ├── 向上抽象
│   └── 向下具体化
├── AutoAssociator (自动联想)
│   ├── On-Insert 联想
│   ├── 扩散激活
│   └── 自发回忆
└── MemoryEngine (存储引擎)

认知层 (Cognition)
├── 意图理解
├── 推理生成
└── 摘要提取
```

## 数据结构

### Episode (记忆单元)

```python
Episode(
    episode_id="doc_001",
    source_type="document",  # document/chat/api
    summary="Deep Learning Survey",
    
    # 层级映射
    concepts=["深度学习", "CNN"],      # Level 2: 中阶概念
    entity_ids=["ResNet", "ImageNet"], # Level 1+: 具体实体
    
    # 自动提取的元数据
    metadata={
        "domain": "人工智能",           # Level 3: 高阶领域
        "hierarchy_depth": 3,
        "parent_doc": None,
    }
)
```

### 知识图谱边类型

```python
EdgeType:
    # 层级边
    CONCEPT_LINK     # Instance → Concept (向上)
    SAME_THEME       # Concept → Instance (向下)
    
    # 联想边
    EPISODE_SIMILARITY  # 相似 Episode
    SEMANTIC_SIMILARITY # 语义相似
    
    # 推断边 (LLM 生成)
    INFERRED_RELATION   # 跨 Episode 关系
    ANALOGOUS_TO        # 结构类比
```

## 竞品对比

| 产品 | 定位 | 核心能力 | 差异 |
|------|------|----------|------|
| Notion | 文档管理 | 结构化笔记 | 无自动联想 |
| Obsidian | 知识管理 | 双向链接 | 需手动建立关联 |
| Roam | 网络笔记 | 块引用 | 无层级抽象 |
| **NeuroAgent** | **智能助手** | **自动联想 KG** | **自动层级化 + 联想浮现** |

**核心差异**：
- Notion/Obsidian：用户手动整理 → 系统存储
- NeuroAgent：用户自然输入 → 系统自动理解、联想、浮现

## 总结

**NeuroAgent** 是面向个人用户的**第二大脑**：

1. **输入自然**：用户只管上传文档，系统自动理解
2. **层级清晰**：高阶概念 → 低阶实例，双向联想
3. **自动关联**：无需手动打标签，系统自动发现关联
4. **智能浮现**：需要时自动联想相关记忆

**技术核心**：层级化 KG + 自动联想
**产品形态**：个人知识助手
**用户价值**：让知识主动找上你，而非你去搜索知识
